# Parte 1 – Fluxo para subir um projeto pela primeira vez



1️⃣ Inicializar git no projeto local
#cd caminho/do/projeto         entra na pasta do projeto
#git init                      inicializa o git
#Cria a pasta .git escondida, que controla o versionamento.



2️⃣ Adicionar arquivos ao git
#git add .          # adiciona todos os arquivos do projeto
                    #Pode adicionar arquivo por arquivo: git add nome_arquivo.



3️⃣ Criar o primeiro commit
#git commit -m "first commit initial project"



4️⃣ Criar o repositório no GitHub
#Vai em GitHub → “New repository”
#Escolhe o nome
#Não marca README, .gitignore ou licença
#Cria o repositório vazio



5️⃣ Conectar o repositório local ao GitHub
#git remote add origin https://github.com/SEU_USUARIO/NOME_DO_REPO.git
#git remote -v                  # checar se a conexão funcionou



6️⃣ Configurar branch principal
#git branch -M main         # renomeia branch principal para main



7️⃣ Subir projeto pro GitHub (push)
#git push -u origin main
#-u vincula a branch local à branch remota, facilitando futuros git push.



8️⃣ Próximos commits
#Sempre que fizer alterações:
#git add .
#git commit -m "Mensagem explicando alteração"
#git push

#se algo estiver fora do commit local e der erro:
#git pull --rebase origin main      Puxa as mudanças do remoto (README ou outra coisa criada manualmente no GitHub) e aplica as suas mudanças por cima

#git log	                        Mostra histórico de commits
#git diff	                        Mostra diferenças entre arquivos modificados e última versão commitada
#git reset	                        Remove arquivo do staged
